# 2026-04-13 — Integração Supabase completa

## Resumo

Integração de todos os dados do app com Supabase. Perfil, planos de treino, sessões e cargas agora são persistidos no banco. Antes, tudo era in-memory via Zustand (perdia ao fechar o app).

## O que foi feito

### Novos arquivos
- `src/types/database.ts` — tipos das rows do banco (snake_case)
- `src/utils/caseMapper.ts` — conversão snake_case ↔ camelCase + datas BR ↔ ISO
- `src/services/profileService.ts` — CRUD do perfil (get, create, update)
- `src/services/workoutService.ts` — CRUD de planos e sessões (inserção multi-tabela sequencial)
- `src/navigation/AppNavigator/AppNavigator.styles.ts` — estilos do loading

### Stores refatorados
- `profileStore.ts` — async com `loadProfile`, `isProfileLoaded`, `isLoading`, `error`, `reset()`
- `workoutStore.ts` — async com `loadPlans`, `loadSessions`, `saveDraft`, `finishWorkout`, `reset()`
- `authStore.ts` — `signOut` agora reseta profile e workout stores

### Telas atualizadas
- `OnboardingScreen` — `handleSave` async, exibe loading/erro
- `SettingsScreen` — `handleSave` async, exibe loading/erro
- `ActiveWorkoutScreen` — `handleFinish` async, inclui `exerciseName` nos weights
- `AddWorkoutExercisesScreen` — `handleSave` async com loading
- `HomeScreen` — `mockMonthlyTotal` substituído por cálculo real
- `AppNavigator` — carrega profile/plans/sessions no startup, mostra loading

### Tipo atualizado
- `ExerciseWeight` — adicionado campo `exerciseName` (necessário para a tabela `workout_session_weights`)

### Bug fix
- Conversão de datas: `DD/MM/YYYY` (formato brasileiro) → `YYYY-MM-DD` (ISO para Supabase)
