# 2026-04-27 — Tema Dark/Light + theming completo

## Resumo

Sistema de tema dark/light totalmente funcional. Paletas refinadas para os dois modos, novos aliases semânticos (`onSecondary`, `onSecondaryMuted`, `textInverse`) e migração arquitetural ampla: todos os arquivos que importavam `colors` direto de `src/constants` (apontando sempre para `darkColors`) foram migrados para `theme.colors` via styled-components / `useTheme()`. Antes da migração, trocar para o modo claro deixava muitos textos pálidos, botões com baixo contraste e o header roxo com texto invisível.

## O que foi feito

### Novos arquivos

- `src/constants/theme.ts` — `themes` (dark/light), `getTheme(mode)`, type `AppTheme`
- `src/stores/themeStore.ts` — Zustand store com persistência AsyncStorage (`@gymtracker:theme-mode`)
- `src/types/styled.d.ts` — augmenta `DefaultTheme` do styled-components

### Paletas refinadas (`src/constants/colors.ts`)

- **Dark** ajustado para tons mais neutros (sem dominância azul-arroxeada antiga); primary `#FACC15`
- **Light** corrigido completamente — antes `background: #dddddd` cinza com elevated `#c9c3c3` (mais escuro que o fundo, hierarquia invertida); agora `#F4F6FA` off-white com cards brancos `#FFFFFF`
- Light primary mudado para `#EAB308` (amarelo vibrante) p/ alta legibilidade do texto escuro em botões
- `textInverse` agora é escuro nos dois temas (primary é amarelo nos dois → texto contrastante deve ser escuro)
- Novos aliases `onSecondary` / `onSecondaryMuted` (sempre claros) p/ texto sobre header roxo (`secondaryDark`) que não troca entre temas

### App.tsx

- `<ThemeProvider theme={appTheme}>` envolvendo o app
- Sincroniza `StatusBar` (`barStyle` + `backgroundColor`) e chama `Appearance.setColorScheme` para que UI nativa (teclado, alerts) acompanhe o tema
- `loadTheme()` chamado no bootstrap antes de renderizar a navegação

### Toggle UI

- Item "Light Mode" / "Dark Mode" no `ProfileMenu` da HomeScreen com ícone sol/lua
- HomeScreen migrada para `useTheme()` (cores reativas)

### Refactor arquitetural — migração colors → theme.colors

Todos os arquivos `.styles.ts` (~25) e os componentes/telas relevantes migrados:

- **Componentes:** Button, Input, ProgressBar, Card, ChipSelect, WorkoutListItem
- **Telas:** Home, Settings, Statistics, WorkoutPlans, AddWorkoutPlan, AddWorkoutExercises, WorkoutDay, ActiveWorkout
- **Subcomponentes:** CustomDayForm, DayChipSelector

### Fixes de contraste

- **Button outline** — interpolação aninhada de template literal estava quebrada (border-color virava código de função). Corrigido para usar `theme` no nível externo
- **Header roxo** em modo claro — `neutral50` é escuro no light, então títulos do header ficavam escuros sobre roxo (ilegível). Migrado para `onSecondary` (sempre claro)
- **Card variant="purple"** — texto interno (Meta Semanal) também migrado para `onSecondary`
- **DayTitle/DaySubtitle do ActiveWorkout** — não estão dentro do header roxo (são uma section abaixo, sobre fundo da página). Usam `text`/`textSecondary` corretamente
- **BottomControlText** (Pause/Finalizar/Zerar) — variant-aware: neutral usa `text`, primary usa `textInverse`, danger usa `onSecondary`. Antes "Pause" ficava branco em fundo cinza-claro
- **Botões primários** — texto/ícone trocados de `theme.colors.background` para `theme.colors.textInverse` (evita texto invisível no botão amarelo do light)
