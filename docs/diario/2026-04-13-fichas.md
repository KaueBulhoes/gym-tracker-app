# 2026-04-13 (Fichas) — Gerenciamento de fichas de treino

## Resumo

Reestruturação completa do gerenciamento de planos de treino. Adicionado conceito de ficha ativa, tela dedicada para gerenciar fichas, nome do plano, descrição por dia, duração com vencimento e alerta.

## Migração SQL

6 novas colunas em `workout_plans`: name, is_active, finished_at, duration_days, expires_at, expiry_dismissed.
1 nova coluna em `workout_plan_days`: description.

## O que foi feito

### Nova tela
- `WorkoutPlansScreen` — lista de fichas com switch de seleção, modal de detalhes, ativação

### Telas modificadas
- `AddWorkoutPlanScreen` — gera nome do plano (Treino ABCDE / Treino Semanal / título custom)
- `AddWorkoutExercisesScreen` — input de duração da ficha em dias
- `WorkoutDayScreen` — input de descrição do dia (ex: "Peito e Bíceps")
- `HomeScreen` — bottom bar "Planos" em vez de "Adicionar", filtra plano ativo, descrição nos dias, alerta de vencimento com modal de renovação

### Service/Store
- `workoutService` — setActivePlan, renewPlan, dismissExpiry, savePlan agora inclui name/active/duration
- `workoutStore` — setActivePlan, renewPlan, dismissExpiry, updateDraftDuration, startDraft aceita name e duration

### Types
- `WorkoutPlan` — name, isActive, finishedAt, durationDays, expiresAt, expiryDismissed
- `WorkoutDay` — description
- `PlanRow`, `DayRow` — colunas correspondentes
