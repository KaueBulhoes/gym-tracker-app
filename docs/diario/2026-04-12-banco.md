# 2026-04-12 — Mapeamento do banco de dados Supabase

## O que foi feito
- Mapeamento completo de todas as tabelas necessárias no Supabase (7 tabelas)
- Criada documentação detalhada de cada tabela em `docs/banco/` (schema, RLS, relacionamentos, mapeamento com TypeScript)
- Gerado SQL de migração com tabelas, índices, RLS e triggers em `docs/banco/migrations/001_create_tables.sql`
- Atualizado CLAUDE.md com `docs/banco/` na estrutura de documentação
- Atualizado `docs/api/index.md` com links para a documentação de cada tabela

## Decisões tomadas
- Documentação de tabelas Supabase agora é etapa padrão do projeto
- Todas as tabelas ficam no schema `public`, conectadas a `auth.users` via `user_id`
- RLS via JOIN para tabelas sem `user_id` direto (workout_plan_days, workout_plan_exercises, exercise_rep_schemes, workout_session_weights)
- `exercise_name` desnormalizado em workout_session_weights para queries de progressão de carga
- `day_name` desnormalizado em workout_sessions para queries rápidas

## Tabelas mapeadas
1. `profiles` — perfil do usuário (onboarding)
2. `workout_plans` — planos de treino
3. `workout_plan_days` — dias de cada plano
4. `workout_plan_exercises` — exercícios de cada dia
5. `exercise_rep_schemes` — blocos de reps (drop sets)
6. `workout_sessions` — sessões finalizadas
7. `workout_session_weights` — cargas por exercício por sessão
